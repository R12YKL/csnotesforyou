Internet & Protocols

Introduction to Networks

A network is a collection of computers and devices connected together to share data and resources.

The Internet is the largest global network, enabling billions of devices to communicate worldwide.

Network Types

LAN (Local Area Network)

  • Covers limited geographic area (home, office, school)
  • Fast data transfer rates
  • Usually owned by single organization
  • Example: School computer lab connected via Ethernet

WAN (Wide Area Network)

  • Covers large geographic areas
  • Slower than LAN due to distance
  • Often owned by telecommunications companies
  • Example: Internet connecting worldwide computers

MAN (Metropolitan Area Network)

  • Covers city-sized area
  • Between LAN and WAN in size and speed
  • Example: City-wide school district network

PAN (Personal Area Network)

  • Very small area (few meters)
  • Wireless connection (Bluetooth)
  • Example: Phone, headphones, and smartwatch

The Internet

The Internet is a worldwide system of interconnected computer networks that communicate using standardized protocols.

Key Characteristics:

  • Decentralized: No single controlling authority
  • Scalable: Can accommodate new devices indefinitely
  • Redundant: Multiple paths between destinations
  • Protocol-based: Uses standardized communication rules

How It Works:

  1. Your device connects to Internet Service Provider (ISP)
  2. ISP provides connection to larger networks
  3. Data travels through multiple routers to destination
  4. Each router forwards data closer to target
  5. Destination device receives data

TCP/IP Model

TCP/IP (Transmission Control Protocol/Internet Protocol) is the fundamental protocol suite governing Internet communication.

TCP/IP Layers:

Application Layer
(HTTP, HTTPS, FTP, SMTP, DNS, SSH)

Transport Layer
(TCP, UDP)

Internet Layer
(IP, ICMP, IGMP)

Link Layer
(Ethernet, WiFi, PPP)

Link Layer (Physical)

  • Hardware addressing (MAC addresses)
  • Physical transmission over cables or wireless
  • Protocols: Ethernet, WiFi, PPP

Internet Layer

  • Routing data across networks
  • IP addresses identify devices
  • Protocols: IP (IPv4, IPv6), ICMP

Transport Layer

  • End-to-end communication between applications
  • Port numbers identify specific services
  • Protocols: TCP (reliable), UDP (fast)

Application Layer

  • Services users directly interact with
  • HTTP for web browsing, SMTP for email, FTP for file transfer
  • Protocols: HTTP, HTTPS, FTP, SMTP, DNS, SSH

IP Addresses

IP (Internet Protocol) addresses uniquely identify devices on networks.

IPv4 (Internet Protocol Version 4)

  • 32-bit address: xxx.xxx.xxx.xxx (e.g., 192.168.1.1)
  • Each segment (octet) ranges from 0-255
  • Supports ~4.3 billion addresses
  • Running out of addresses due to Internet growth

IPv6 (Internet Protocol Version 6)

  • 128-bit address: 2001:0db8:85a3:0000:0000:8a2e:0370:7334
  • Supports ~340 undecillion addresses
  • Gradually replacing IPv4
  • Better security and efficiency

Public vs Private IP:

  • Public IP: Unique on entire Internet, assigned by ISP
  • Private IP: Used within LAN, not routable on Internet
  • Example private range: 192.168.0.0 to 192.168.255.255

DNS (Domain Name System)

DNS translates human-readable domain names into IP addresses.

How DNS Works:

  1. User enters domain name in browser (www.google.com)
  2. Browser queries DNS resolver (usually provided by ISP)
  3. Resolver queries root nameserver
  4. Root directs to TLD (Top-Level Domain) nameserver
  5. TLD directs to authoritative nameserver
  6. Authoritative nameserver returns IP address
  7. Browser connects to web server at that IP

Domain Name Structure:

www.example.com

.com = Top-Level Domain (TLD)
example = Second-level domain
www = Subdomain

Common TLDs:

  • .com (Commercial)
  • .org (Organization)
  • .edu (Educational)
  • .gov (Government)
  • .ie (Ireland)
  • .uk (United Kingdom)

Key Internet Protocols

HTTP/HTTPS (HyperText Transfer Protocol)

  • Used for web browsing
  • HTTP: Unencrypted (insecure)
  • HTTPS: Encrypted with TLS/SSL (secure)
  • Request-response based

FTP (File Transfer Protocol)

  • Used for transferring files over network
  • Requires username and password
  • SFTP: Secure version with encryption

SMTP (Simple Mail Transfer Protocol)

  • Used for sending emails
  • Works with mail servers
  • SMTPS: Secure version with encryption

POP3/IMAP (Email Retrieval)

  • POP3: Downloads emails to local device
  • IMAP: Keeps emails on server, accesses them
  • Used by email clients

SSH (Secure Shell)

  • Secure remote login to computers
  • Encrypted communication
  • Used by system administrators

TCP vs UDP

Feature TCP UDP
Reliability Guaranteed delivery No guarantee
Speed Slower (more overhead) Faster
Connection Connection-oriented Connectionless
Use Cases Email, Web, File Transfer Video streaming, Gaming, VoIP

Cybersecurity Basics

Common Threats:

  • Malware: Malicious software (viruses, worms, trojans)
  • Phishing: Fraudulent emails posing as legitimate sources
  • Ransomware: Software that encrypts files for extortion
  • DDoS: Overwhelming servers with traffic
  • Password attacks: Guessing or stealing passwords

Security Best Practices:

  • Use strong, unique passwords
  • Enable two-factor authentication
  • Keep software updated
  • Use HTTPS on websites
  • Don't click suspicious links
  • Regular backups of important data
  • Use antivirus and firewall software

Encryption:

  • TLS/SSL: Protects web communications
  • Scrambles data so only recipient can read it
  • Indicated by HTTPS and lock icon in browser

Key Revision Points

  • Networks connect devices to share data and resources
  • Internet is largest global network
  • TCP/IP is fundamental protocol suite with 4 layers
  • IP addresses uniquely identify devices
  • DNS translates domain names to IP addresses
  • Multiple protocols handle different applications (HTTP, FTP, SMTP)
  • TCP ensures reliability; UDP prioritizes speed
  • Cybersecurity is essential for protecting data online
