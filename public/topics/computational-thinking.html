Computational Thinking

What is Computational Thinking?

Computational Thinking (CT) is a problem-solving approach that breaks down complex problems into manageable pieces, identifies patterns, abstracts away unnecessary details, and designs step-by-step solutions (algorithms).

It's not just about programming—it's a way of thinking that applies to any field where problems need to be solved.

The Four Pillars of Computational Thinking

1. Decomposition

Breaking problems into smaller, manageable subproblems that are easier to understand and solve.

2. Pattern Recognition

Identifying similarities and patterns within problems to find efficient solutions and reuse knowledge.

3. Abstraction

Focusing on the essential features of a problem while ignoring irrelevant details to simplify solutions.

4. Algorithm Design

Creating step-by-step procedures that solve the problem systematically and can be implemented as programs.

Decomposition

Breaking a large, complex problem into smaller, more manageable subproblems.

Why Decompose?

  • Makes problems easier to understand
  • Allows different people to work on different parts
  • Easier to debug and test individual components
  • Reduces cognitive overload

Example: Creating a Game

Main Problem: Develop a video game

Decomposed into:
• Graphics rendering system
• Input handling (keyboard, mouse)
• Game physics engine
• Sound system
• User interface/menus
• Collision detection
• Score/progress tracking

Real-World Applications:

  • Software development: Breaking projects into modules
  • Business processes: Dividing work among departments
  • Scientific research: Breaking investigations into experiments

Pattern Recognition

Identifying recurring patterns, similarities, and themes that help solve new problems efficiently.

How Patterns Help:

  • Avoid "reinventing the wheel" for similar problems
  • Use proven solutions for known patterns
  • Speed up problem-solving by recognizing familiar situations
  • Enable generalization of solutions

Common Programming Patterns:

  • Loops: Repeating actions multiple times
  • Conditionals: Making decisions based on conditions
  • Functions: Reusable blocks of code
  • Data structures: Organizing related data (lists, dictionaries)

Example: Data Search

Finding a contact in your phone:
• Linear search: Check each contact one by one
• Binary search: Jump to middle, narrow down range (faster!)

Pattern recognized: We can use binary search for ANY sorted list

Abstraction

Focusing on the essential features of a problem while hiding or ignoring unnecessary complexity.

Why Abstract?

  • Reduces complexity to manageable levels
  • Focuses attention on relevant details
  • Enables reuse at different levels of detail
  • Makes systems easier to understand and modify

Layers of Abstraction

Using a web browser:
• User level: Click a link, page loads
• Browser level: HTML/CSS/JavaScript rendering
• Network level: HTTP requests, TCP/IP packets
• Hardware level: Electrical signals

Each layer abstracts away complexity below it

Abstraction in Practice:

  • APIs hide implementation details
  • Classes encapsulate data and methods
  • Operating systems abstract hardware
  • Maps abstract real-world geography

Algorithm Design

Creating a precise, step-by-step procedure to solve a problem that can be implemented as a program.

Characteristics of Good Algorithms:

  • Correct: Produces the right answer for valid inputs
  • Efficient: Runs quickly with minimal resources
  • Clear: Easy to understand and implement
  • Finite: Eventually terminates
  • General: Works for a range of inputs

Representing Algorithms

Algorithms can be expressed as:

  • Pseudocode: English-like statements that resemble programming
  • Flowcharts: Visual diagrams showing process flow
  • Code: Actual programming language implementation

Example Algorithm: Recipe

Algorithm to make tea:
1. Fill kettle with water
2. Switch on kettle
3. Wait until water boils
4. Put tea bag in cup
5. Pour hot water into cup
6. Wait 3-5 minutes
7. Remove tea bag
8. Add milk and sugar (optional)
9. Stir and drink

Example: Complete Problem Using CT

Problem: Sort a list of student names alphabetically

Step 1: Decomposition

  • Get the list of names
  • Compare names pairwise
  • Swap if needed
  • Repeat until sorted
  • Display sorted list

Step 2: Pattern Recognition

  • This is a sorting problem
  • Similar to arranging books on a shelf
  • Could use bubble sort, insertion sort, or quick sort

Step 3: Abstraction

  • Focus on the sorting algorithm, not data source
  • Ignore how names are pronounced or spelled
  • Only care about alphabetical order

Step 4: Algorithm

Bubble Sort Pseudocode:
FOR i = 1 TO length(list)-1
  FOR j = 1 TO length(list)-i
    IF list[j] > list[j+1]
      SWAP list[j] and list[j+1]
    END IF
  END FOR
END FOR

Computational Thinking in Different Domains

  • Biology: Sequence analysis, protein folding
  • Medicine: Diagnostic algorithms, treatment planning
  • Business: Process optimization, data analysis
  • Geography: Route planning, resource distribution
  • Music: Composition algorithms, audio processing
  • Art: Computer graphics, generative art

Key Revision Points

  • Computational Thinking is a universal problem-solving approach
  • The four pillars: decomposition, pattern recognition, abstraction, algorithms
  • These techniques work together to solve complex problems
  • CT applies beyond programming to many real-world situations
  • Master each pillar individually, then combine them
