Databases

Introduction to Databases

A database is an organized collection of structured data stored and accessed electronically. Databases are essential for storing and managing large amounts of information efficiently.

Most applications you use daily rely on databases: social media, email, banking, online shopping, weather apps, and more.

Flat Files vs. Relational Databases

Flat Files

Data stored in simple text files (CSV, plain text, etc.).

Flat File Advantages:

  • Simple to understand and create
  • No special software required to view
  • Portable across systems

Flat File Disadvantages:

  • Difficult to manage large amounts of data
  • Data redundancy (repetition)
  • Slow searching and sorting
  • Limited query capabilities
  • Vulnerability to data inconsistency

Example Flat File:

students.csv
StudentID, Name, Email, Class
001, John Smith, john@school.ie, 5A
002, Mary Jones, mary@school.ie, 5A
003, Peter Brown, peter@school.ie, 5B

Relational Databases

Data organized into related tables with structured relationships.

Relational Database Advantages:

  • Reduces data redundancy
  • Fast data retrieval
  • Maintains data integrity
  • Supports complex queries
  • Multiple users can access simultaneously
  • Better security and backup capabilities

Relational Database Disadvantages:

  • More complex to design
  • Requires specialized software
  • Can be slower for very simple queries
  • Steeper learning curve

Key Database Concepts

Table (Relation): A two-dimensional structure with rows and columns.

Record (Tuple): A single row in a table representing one instance of the entity.

Field (Attribute): A column in a table representing a property of the entity.

Example Table:

StudentID Name Email Class
001 John Smith john@school.ie 5A
002 Mary Jones mary@school.ie 5A
003 Peter Brown peter@school.ie 5B

Primary Key: A field (or combination of fields) that uniquely identifies each record. No two records can have the same primary key.

Example Primary Key:

In the Students table above, StudentID is the primary key because each student has a unique ID.

Foreign Key: A field in one table that references the primary key of another table, creating relationships between tables.

Example Foreign Key:

Enrollments Table:
EnrollmentID | StudentID | CourseID | Grade
1001 | 001 | C101 | A
1002 | 001 | C102 | B
1003 | 002 | C101 | A+

StudentID is a foreign key linking to the Students table

Entities and Entity-Relationship Diagrams

Entity: A real-world object or concept that we want to store information about (e.g., Student, Course, Teacher).

Entity-Relationship (E-R) Diagram: A visual representation showing entities and their relationships.

Relationship Types:

  • One-to-One (1:1): One entity relates to exactly one other entity
  • One-to-Many (1:M): One entity relates to many instances of another entity
  • Many-to-Many (M:M): Many entities relate to many other entities

Example Relationships:

School Database:

• One Teacher has Many Students (1:M)
• One Student has Many Teachers (M:M)
• One Student has One Homeroom Class (1:1)
• One Class has Many Students (1:M)

Data Types and Field Properties

Common Data Types:

Data Type Description Example
INT/INTEGER Whole numbers 100, -5, 0
VARCHAR/TEXT Text strings 'John Smith', 'Email'
DECIMAL/FLOAT Numbers with decimals 3.14, 99.99
DATE/DATETIME Date and/or time 2024-01-15, 14:30:00
BOOLEAN True/False values TRUE, FALSE

Field Constraints:

  • NOT NULL: Field must have a value
  • UNIQUE: All values must be different
  • PRIMARY KEY: Uniquely identifies records
  • FOREIGN KEY: References another table's primary key
  • DEFAULT: Automatic value if none provided

Normalization

Normalization is the process of organizing database tables to reduce redundancy and improve data integrity.

First Normal Form (1NF):

  • All fields contain atomic (indivisible) values
  • No repeating groups of columns

Second Normal Form (2NF):

  • Meet 1NF requirements
  • All non-key fields depend on the entire primary key

Third Normal Form (3NF):

  • Meet 2NF requirements
  • No non-key field depends on another non-key field

Example of Normalization:

Before Normalization (Not 1NF):
StudentID | Name | Courses
001 | John | Math, English, Science

After Normalization (1NF):
Separate into two tables:
Students: StudentID, Name
Enrollments: StudentID, CourseID

Introduction to SQL

SQL (Structured Query Language) is the standard language for managing databases.

Basic SQL Operations:

SELECT: Retrieve data from tables

SELECT Name, Email FROM Students WHERE Class = '5A';

INSERT: Add new records

INSERT INTO Students (StudentID, Name, Email, Class)
VALUES (004, 'Sarah Lee', 'sarah@school.ie', '5C');

UPDATE: Modify existing records

UPDATE Students SET Email = 'newemail@school.ie'
WHERE StudentID = 001;

DELETE: Remove records

DELETE FROM Students WHERE StudentID = 003;

Database Management Systems

Common DBMS Software:

  • MySQL: Open-source, widely used for web applications
  • PostgreSQL: Advanced open-source database
  • Microsoft SQL Server: Enterprise-level database
  • Oracle Database: Large-scale enterprise database
  • SQLite: Lightweight, embedded database

Database Design Process

  1. Requirements Analysis: Understand what data needs to be stored
  2. Conceptual Design: Create E-R diagrams showing entities and relationships
  3. Logical Design: Convert E-R diagrams to table structures
  4. Normalization: Apply normalization rules to reduce redundancy
  5. Physical Design: Implement in actual DBMS software
  6. Testing and Maintenance: Test functionality and optimize as needed

Key Revision Points

  • Databases store and organize large amounts of structured data
  • Relational databases use tables with relationships between them
  • Primary keys uniquely identify records; foreign keys create relationships
  • Normalization reduces redundancy and improves data integrity
  • SQL is the standard language for database operations
  • Good database design requires planning and understanding of relationships
