Computer Systems

The Von Neumann Architecture

The modern computer is based on the Von Neumann Architecture, proposed by John von Neumann in 1945. This fundamental design principle is still used in today's computers.

Key Components:

  • Central Processing Unit (CPU): Executes instructions and performs calculations
  • Memory: Stores programs and data (RAM and storage)
  • Input Devices: Allow user interaction (keyboard, mouse, sensors)
  • Output Devices: Display or transmit results (screen, speakers, printers)
  • Control Unit: Directs the flow of data and instructions
  • Arithmetic Logic Unit (ALU): Performs calculations and logical operations
  • Buses: Communication pathways for data transfer

The Fetch-Decode-Execute Cycle

Every program execution follows this fundamental cycle:

  1. Fetch: The CPU retrieves an instruction from memory
  2. Decode: The control unit interprets the instruction
  3. Execute: The CPU carries out the operation

This cycle repeats billions of times per second in modern computers.

The CPU: Heart of the Computer

CPU Functions:

  • Execute program instructions
  • Perform arithmetic and logic operations
  • Control the flow of data between components
  • Manage timing and synchronization

CPU Performance Factors:

Factor Description Impact
Clock Speed Measured in GHz (cycles per second) Higher speed = faster execution
Number of Cores Multiple processors in one CPU Enables parallel processing
Cache Size Fast memory close to the CPU Reduces memory access time
Bus Width Amount of data transferred at once Wider bus = more data per cycle

Registers

Registers are tiny storage locations inside the CPU that hold data currently being processed. Common registers include:

  • Accumulator: Stores results of arithmetic operations
  • Program Counter: Points to the next instruction to execute
  • Memory Address Register (MAR): Holds the address of data to fetch
  • Memory Data Register (MDR): Holds data being transferred to/from memory

Memory Systems

RAM (Random Access Memory)

RAM is the main working memory of a computer:

  • Volatile: loses all data when power is turned off
  • Fast access: nanosecond response times
  • Measured in GB
  • Used for running programs and storing temporary data
  • More RAM allows more programs to run simultaneously

Storage Devices

  • Hard Disk Drives (HDD): Mechanical, slower, cheaper, large capacity
  • Solid State Drives (SSD): No moving parts, faster, more reliable, more expensive
  • USB Flash Drives: Portable, limited write cycles
  • Cloud Storage: Remote access, scalable capacity

Cache Memory

Cache is ultra-fast memory that stores frequently accessed data:

  • L1 Cache: Fastest, smallest (inside CPU)
  • L2 Cache: Larger than L1, slower
  • L3 Cache: Larger than L2, shared among cores

Buses and Data Transfer

Buses are communication highways that connect computer components:

Types of Buses:

  • Address Bus: Carries memory addresses (unidirectional)
  • Data Bus: Carries actual data (bidirectional)
  • Control Bus: Carries control signals (unidirectional)

Bus Width

The number of wires/bits a bus can carry simultaneously affects data transfer speed. A 64-bit bus can transfer 8 bytes at once, while a 32-bit bus transfers 4 bytes.

Common Interfaces:

  • USB (Universal Serial Bus): Standard for peripherals
  • PCIe (PCI Express): High-speed connection for graphics cards, SSDs
  • SATA: Connection standard for storage devices
  • Bluetooth: Wireless communication

Operating System Functions

The Operating System (OS) is the master program that manages all computer resources:

  • Process Management: Manages running programs and allocates CPU time
  • Memory Management: Allocates and deallocates RAM
  • File Management: Organizes and manages data storage
  • Device Management: Controls input/output devices
  • User Interface: Provides way for users to interact with the computer
  • Security and Access Control: Protects system resources and user data

Common Operating Systems:

  • Windows: Desktop/laptop, user-friendly
  • macOS: Apple computers, Unix-based
  • Linux: Open-source, widely used in servers
  • iOS/Android: Mobile operating systems

Hardware Components Summary

Component Function Key Points
Motherboard Main circuit board connecting all parts Houses CPU, RAM, chipset
Power Supply Provides electrical power Measured in watts; reliability is critical
Graphics Card Processes and displays graphics Essential for gaming and video editing
Network Card Enables internet/network connection Ethernet or WiFi; often built into motherboard
Cooling System Removes excess heat Air or liquid cooling; prevents overheating

Key Revision Points

  • Von Neumann Architecture remains the basis for modern computers
  • CPU executes instructions through Fetch-Decode-Execute cycle
  • Memory hierarchy: CPU cache > RAM > Storage
  • Buses enable communication between components
  • Operating System manages all computer resources
  • Multiple factors affect computer performance
