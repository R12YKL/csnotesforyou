Software Development Process

Introduction to SDLC

The Software Development Life Cycle (SDLC) is a structured process for planning, creating, testing, and deploying software applications.

Following a systematic approach ensures quality, reduces errors, manages costs, and meets user expectations.

Stages of SDLC

1. Planning and Requirements

  • Define project scope and objectives
  • Identify stakeholders and their needs
  • Assess feasibility and resources required
  • Create project timeline and budget
  • Document functional and non-functional requirements

2. Analysis

  • Study system requirements in detail
  • Identify potential problems and constraints
  • Create system specifications
  • Document data flow and processes
  • Plan system architecture

3. Design

  • Create system design and architecture
  • Design database structure
  • Plan user interface layout
  • Create detailed design documents
  • Plan security measures

4. Development/Implementation

  • Write the actual code
  • Follow coding standards
  • Use version control for code management
  • Document code with comments
  • Integrate components regularly

5. Testing

  • Unit testing: Test individual components
  • Integration testing: Test components working together
  • System testing: Test complete system
  • User acceptance testing: User validates requirements
  • Document and fix bugs

6. Deployment

  • Release software to production environment
  • Install and configure systems
  • Train users
  • Migrate data if needed
  • Monitor initial performance

7. Maintenance and Support

  • Fix bugs discovered in production
  • Provide user support
  • Make updates and improvements
  • Ensure system security
  • Plan for future enhancements

Waterfall Model

The Waterfall Model is a traditional, linear SDLC approach where each phase flows into the next.

Waterfall Characteristics:

  • Sequential: Phases completed one after another
  • No iteration: Difficult to return to previous phases
  • Documentation-heavy: Extensive documentation required
  • Predictable: Timeline and costs easier to forecast

Waterfall Advantages:

  • Clear structure and milestones
  • Good for well-defined projects
  • Suitable for fixed-price contracts
  • Easy to manage large teams

Waterfall Disadvantages:

  • Inflexible to changing requirements
  • Problems discovered late in development
  • Users see product only at end
  • Can result in wasted effort

Best For:

Projects with clear, fixed requirements (e.g., government contracts, safety-critical systems)

Agile Development

The Agile Model emphasizes iterative development, continuous feedback, and flexibility.

Agile Principles:

  • Customer collaboration over comprehensive documentation
  • Responding to change over fixed plans
  • Individuals and interactions over processes
  • Working software over extensive documentation

Agile Process (Scrum):

  1. Sprint Planning: Select work for 1-4 week sprint
  2. Daily Standup: 15-minute team synchronization
  3. Development: Team works on selected tasks
  4. Sprint Review: Demonstrate completed work to stakeholders
  5. Retrospective: Team reflects and improves process

Agile Advantages:

  • Flexible to changing requirements
  • Regular working software deliverables
  • Early problem detection
  • Better team communication
  • Faster time to market

Agile Disadvantages:

  • Requires experienced team
  • Less predictable budget/timeline
  • Can lack comprehensive documentation
  • Depends on customer availability

Best For:

Projects with evolving requirements, innovation-focused work, fast-moving environments

Testing Types

Unit Testing

  • Testing individual functions or methods
  • Done by developers during development
  • Catches bugs early
  • Example: Testing a password validation function

Integration Testing

  • Testing components working together
  • Checks data flow between modules
  • Identifies interface issues
  • Example: Testing payment and inventory systems together

System Testing

  • Testing complete integrated system
  • Validates end-to-end functionality
  • Tests non-functional requirements (performance, security)
  • Example: Testing entire e-commerce system

User Acceptance Testing (UAT)

  • Users test system against requirements
  • Validates business requirements are met
  • Final approval before deployment
  • Example: Client reviewing custom software

Other Testing Types:

  • Performance Testing: Checks speed and scalability
  • Security Testing: Identifies vulnerabilities
  • Regression Testing: Ensures fixes don't break existing features
  • Load Testing: Tests system under heavy usage

Development Team Roles

Project Manager

  • Overall project coordination
  • Manages timeline, budget, resources
  • Communicates with stakeholders

Business Analyst

  • Gathers and analyzes requirements
  • Bridges business and technical teams
  • Documents specifications

Software Developer/Programmer

  • Writes code according to specifications
  • Performs unit testing
  • Collaborates with other developers

Software Architect

  • Designs overall system structure
  • Makes technology decisions
  • Ensures scalability and reliability

Quality Assurance (QA) Tester

  • Performs testing to find bugs
  • Documents test cases and results
  • Validates requirements are met

DevOps Engineer

  • Manages deployment and infrastructure
  • Automates testing and deployment
  • Monitors system performance

Version Control

Version Control systems track code changes and enable collaboration.

Benefits:

  • Maintains history of all changes
  • Allows reverting to previous versions
  • Enables multiple developers to work together
  • Prevents code conflicts
  • Provides code review capability

Common Systems:

  • Git: Most widely used distributed version control
  • GitHub: Cloud-based Git repository
  • SVN: Centralized version control

Software Quality Metrics

Code Coverage

Percentage of code tested by automated tests. Higher coverage = fewer untested paths.

Bug Density

Number of bugs per 1000 lines of code. Lower is better.

Technical Debt

Cost of fixing shortcuts and poor decisions made during development.

Cyclomatic Complexity

Measure of code complexity. Lower values indicate simpler, more maintainable code.

Key Revision Points

  • SDLC provides structured approach to software development
  • Waterfall: Sequential, fixed requirements
  • Agile: Iterative, flexible, customer-focused
  • Testing is critical at multiple levels
  • Teams include multiple specialized roles
  • Version control essential for team collaboration
  • Quality metrics help measure software success
