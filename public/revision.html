Revision Notes

Quick reference guide covering key concepts across all topics.

History of Computing

  • E.W Dijkstra Quote: "When all is said and done, the only thing computers can do for us is to manipulate symbols and produce results of such manipulations."
  • Computer: A device that can be programmed to carry out general purpose logical/mathematical computations.
  • Programmable Computer: Can be re-programmed to improve solutions, or adapted when the problem changes.
  • General Purpose Computer: Can carry out a wide range of tasks, not built for a specific purpose.
  • Digital Computer: One that operates using digital signals.
  • Signals: Used to represent changing information. Discrete, can only have a limited number of values.
  • Electronic Computer: A computer that operates through the use of electricity.
  • Electricity: The movement of electrons through a material.
  • Semiconductor: A material that falls somewhere between a conductor and an insulator. E.g. Silicon. They can be used to control the flow of electricity making modern computers possible.
  • Modern Computers:
    • Able to carry out general purpose computation via programs.
    • Fairly inexpensive, robust and portable.
    • Have good performance.
    • Relatively good energy efficiency.
    • Fairly durable.
  • Abacus: Aid to a person doing computations.
  • Computers (People): As early as the 17th century the term Computers referred to people doing mathematical calculations.
  • Slide Rule: 1600s analog computer. Able to calculate logarithms, trigonometric functions, squares, and cubes with their roots.
  • Difference Engine (1822): Charles Babbage designed a mechanical computer used to calculate polynomials. Solved the problem of human error in computing. Size of a car, thousands of moving parts.
  • Analytical Engine: Charles Babbage, general purpose computer able to store data on punched cards. Programs stored on punched cards. Never built.
  • Ada Lovelace: First computer programmer. Realized the Analytic Engine could be used for general purpose computing. Wrote the first algorithm designed to be carried out by a machine.
  • Percy Ludgate: Irish, designed a similar machine to Babbage's Analytical Engine, smaller and more portable. Also never built.
  • Turing Machine: Alan Turing's A-machine (automatic machine). Demonstrated the true power of computing before modern computers existed. Consists of:
    • A tape that symbols can be written to and read from.
    • A head that does the reading and writing.
    • A state register that stores the state of the machine at any given time.
    • A table of instructions that tells the machine what to do next.
  • Vacuum Tubes: Small devices that control the flow of electricity.
  • ENIAC: Electronic Numerical Integrator and Computer. First electronic, digital, programmable, general purpose computer. Kathleen McNulty, an Irish woman, was one of the first programmers to work on it.
  • Transistors: Small electronic switches, performed the same job as a vacuum tube but more efficiently.
  • Integrated Circuits: Third Generation of Computers, a number of electronic circuits on one semiconductor, a Chip. Example: AGS (Apollo Guidance System).
  • Microprocessors: Fourth Generation, CPUs. 1971, Intel 4004.
  • Personal Computers: PCs made possible by microprocessors with billions of transistors. Commodore, Apple, IBM. Development of OS such as Windows.
  • Generations:
    • First Generation: Vacuum Tubes.
    • Second Generation: Transistors.
    • Third Generation: Integrated Circuits.
    • Fourth Generation: Microprocessors.
    • Potential Fifth Generation: Quantum Computing (Far future).
  • Ireland and Supercomputers: 14 of the 500 fastest computers on Earth in Ireland. One named Kay after Kathleen McNulty.

Computer Systems

  • Computer System: A system that involves interaction between hardware, software, data and the user.
  • Computer Architecture: Specification of how a computer's parts are physically and logically laid out and connected.
  • Von Neumann Architecture (1945): Comprised of four main parts:
    • Central Processing Unit (CPU) which processes instructions.
    • Memory that stores data and instructions.
    • A bus connecting CPU to memory.
    • Input and output devices.
  • CPU: Carries out computations on data by following instructions. Contains ALU, control unit and registers.
  • ALU: Arithmetic and logic unit where arithmetic and logical operations take place.
  • Registers: Temporary memory that the CPU can access very quickly. Examples: Programme Counter, Data Registers, Address Registers.
  • Programme Counter: A register that stores where the CPU is in the programme sequence and contains the memory location of the next instruction.
  • Address Registers: Store memory addresses.
  • RAM: Random access memory. Volatile - temporary memory that holds data and instructions of the program currently being executed.
  • Volatile Memory: When the computer loses power the data is lost.
  • ROM: Read only memory, non-volatile. Used to store the BIOS.
  • BIOS: Basic input/output system. Stores basic startup routines and performs necessary checks before handing control to the OS.
  • Fetch and Execute Cycle: CPU fetches instruction from memory, decodes it, executes it, and returns the result to memory if required.
  • Clock: Generates a signal of continuous electrical pulses at a stable frequency.
  • CPU Speed: Mainly determined by clock speed. 5GHz = 5 billion instructions per second. Also affected by number of cores.
  • Cores: Parts of the CPU that execute instructions simultaneously.
  • Bus: Connects the CPU and Memory. Communication channel that carries data between computer components.
  • Input and Output Devices: Examples include display, secondary storage, USBs, mice.
  • Secondary Storage: Stores data persistently. Slower but cheaper than primary storage.
  • Hard Drives: Two main types: Magnetic and SSD.
  • Magnetic Drive: Has rotating magnetic "platter" with data read by "head". Cheaper than SSDs, holds more storage.
  • SSD: Solid state drive, faster and more durable than magnetic drives. No moving parts. Based on flash memory.
  • Motherboard: Circuit board that parts are physically mounted on.
  • Circuit Board: Board of non-conductive material that contains pathways connecting devices.
  • Binary: Symbol pairs (1,0). Used to represent all information on a computer.
  • Operating System: Series of programs that control the operation of the computer. Functions include managing I/O, interactions with other computers, memory management, UI, file system, hardware, and security.
  • Layer Model: User interacts with program, program interacts with OS, OS interacts with hardware.
  • WWW (1989): World Wide Web by Tim Berners-Lee. System of connected files hosted on the internet.
  • Internet (1969): Two computers interacted in first instance of real-time data sharing. Today billions of computers connected via various media.
  • IOT: Internet of Things. Non-computer objects such as cars and TVs becoming interconnected.
  • Client-Server Model: Client issues request to server, server responds. Distributed system with workloads shared across multiple computers.
  • Communication Protocols: Rules describing how to transmit data across a network. Layers: Application (HTTP/HTTPS), Transport (TCP), Network (IP), Physical (WiFi).
  • HTTP: Hypertext Transport Protocol. Request-response protocol allowing clients to request resources.
  • HTTPS: Hypertext Transport Protocol Secure. Automatically encrypts data.
  • URL: Uniform Resource Locator. Format of addressing resources on the internet.
  • TCP: Transmission Control Protocol. Transport layer. Ensures binary is transported reliably via packets.
  • IP: Internet Protocol. Network layer. Handles creation of packets and routing. Every connected computer has unique IP address.
  • WiFi: Wireless Fidelity. Radio wave technology allowing wireless communication and internet access.
  • VOIP: Voice Over Internet Protocol. Delivers voice and video data across the Internet. E.g. Skype, Facetime.
  • MAC Address: Media access control address. Unique hardware identification for each device on a network.
  • DNS: Domain Name Server. Identifies website names by converting IP addresses into readable domain names.
  • Internet Hardware Components: Gateways, Routers, Network Bridges, Switches, Repeaters.
  • Gateways: Connect two networks with different protocols.
  • Routers: Forward data packets between computer networks.
  • Network Bridges: Create a single network from multiple segments.
  • Switches: Forward data to only devices that need to receive it.
  • Repeaters: Receive data and re-transmit it.
  • Cloud Computing: Storing, processing and accessing data over the Internet on servers designed for this purpose. Provides elasticity - pay only for what you use.

Software Development

  • Agile Approach: Flexible and collaborative way of working on projects.
  • Waterfall Approach: More formal and less flexible way of working on projects.
  • Steps of Software Development:
    • Investigate: Identifying and analyzing the nature of the problem.
    • Plan: Planning what to do, timeframes, staff, and management.
    • Design: Top-down or modular approaches.
    • Create: Writing the actual code.
    • Evaluate: Testing the software.
    • Iterate: Improving on the original design.
    • Document: Creating documentation on the project.
  • Design Approaches:
    • Top-down: Starts with main system and breaks into smaller units.
    • Modular: Similar to top-down but each task is a "module" a team works on.
  • Teams:
    • Business Analyst: Identifies users and stakeholders.
    • Project Manager: Forms and supervises the team.
    • System Analyst: Establishes hardware and software requirements.
    • Designer: Designs the software.
    • Developer: Develops the code.
    • Tester: Ensures software functions as expected.
    • Admin Support: Organizes meetings and correspondence.
    • User Liaison: Ensures communication between users and team.

Computational Thinking

  • Computational Thinking: A way of approaching problems particularly useful in computing.
  • Pattern Matching: Identifying patterns and recognizing when two patterns are similar.
  • Abstraction: Ignoring irrelevant details and reducing a problem to its essential features.
  • Decomposition: Breaking a complex task down into smaller and simpler tasks.
  • Algorithms: Using a sequence of steps to solve a problem.
  • Binary: A system of storing data using 1s and 0s.
  • Hex: A base16 system of storing data (0 to F).

Python

  • Python: General purpose language used to solve various problems.
  • Variable Types:
    • String: Alphanumeric characters - "abcde"
    • Int (integer): Whole number - 1
    • Float: Number with decimal point - 1.2
    • Boolean: True/False, 0/1
  • Assigning Variables: Use snake_case by convention. Can't use Python keywords, start with number, contain spaces or special characters.
  • Type Function: Use type() to print out the variable type.
  • Swapping Variables: a,b = b,a (Canonical swap)
  • Operators: +, -, *, /, // (floor divide), % (modulus), ** (exponent)
  • Incrementing: x+=1, x-=1, x*=1, x/=1
  • Rounding: round(x, 2) rounds x to two decimal places.
  • String Concatenation: Joins strings using + operator.
  • String Replication: Multiplies a string using * operator.
  • ASCII: American Standard Code for Information Interchange - A system of encoding characters.
  • ord(): Returns the decimal value of a character.
  • chr(): Converting decimal value into a character.
  • Case Conversion: x.upper(), x.lower()
  • String Length: len(x)
  • Indexing: Starts at [0]. Last value: [-1]. Substring: [6:10]
  • Inputs: input("Input an example: ")
  • Boolean Logic: And, Or, Not - Returns True or False.
  • Comparative Operators: ==, !=, <, >, <=, >=
  • Conditionals: If, Elif, Else
  • Nested Conditionals: If statement inside another if statement.
  • Iteration: Repeating a task using while or for loop.
  • While Loop: while i < 10: repeats code while condition is true.
  • Infinite Loop: while True: with break to exit.
  • For Loop: for i in range(6): repeats code 6 times.
  • For Loop with Range: for i in range(2,100,2) starts at 2, goes to 100, uses every other number.
  • Nested Loops: Two loops, one inside the other.
  • Lists: example_list = [x,y,z]
  • List Methods: append(), clear(), copy(), count(), extend(), index(), insert(), pop(), remove(), reverse(), sort()
  • Testing Software: Check if code works as expected. Record in table: test case ID, description, test data, expected result, actual result, pass(Y/N).
  • Equivalence Partitions: Partition input ranges into areas that behave similarly.
  • Extreme Values: Identify possible extreme values in testing.
  • Boundary Values: Values between two outcomes where bugs are often found.
  • Error Anticipation: Predict possible errors that may occur.
  • Unit Testing: A function that tests if the original function outputs the expected result.
  • Functions: A block of code callable from any other part of the code.
  • Argument: A value(s) taken in by a function.
  • Importing Functions: import filename.py
  • Recursion: Breaking a problem into smaller pieces.
  • Searching Algorithms:
    • Linear: Iterates through each item until it finds the right one.
    • Binary Search: Requires sorted list. Finds halfway index and searches through new smaller list.
  • Sorting Algorithms:
    • Selection Sort: Takes first value, finds smallest in unsorted section, swaps it.
    • Insertion Sort: Same as selection but starts at second and works right.
    • Bubble Sort: Checks two adjacent items and swaps if necessary.
    • Quicksort: Divide and conquer approach.

Modelling

  • Computer Modelling: Any activity that involves using software abstractions to represent a problem.
  • Model: A block of code that takes argument(s) and returns output(s).
  • Scenario: A condition/situation in the environment that can change.
  • Simulation: Running a model to allow different scenarios to be tested.
  • Agent Based Modelling: A model that simulates actions and interactions of individual agents within an environment.
  • Emergent Behaviours: Behaviours examined in the environment as a whole but not within individual agents.

AI and Machine Learning

  • Machine Learning: The use of computers to identify patterns in data and make decisions based on this data.
  • Deep Learning: A subset of machine learning that aims to create algorithms mimicking the human brain.
  • Supervised Learning: Learning a function that maps input to output based on labelled datasets.
  • Unsupervised Learning: Algorithms that find patterns in unlabelled data.
  • Semi-Supervised Learning: Algorithms using combination of labelled and unlabelled data.
  • Reinforcement Learning: Algorithms that learn by interacting with environment using reward and punishment system.
  • Data Quality: Poor data quality leads to inaccurate outputs. Remove missing values, outliers and imbalanced data.

Databases

  • Database: A way of organizing data/information so that it can be accessed easily.
  • Benefits: Store large amounts of data, hold data in organized way, faster searches, freely available, interact with different systems.
  • Flat File Databases: Simplest database, single table.
  • Attributes: Column name and the data it contains (fields).
  • Records: One entry in the table (instance).
  • Data Types: Varchar, Number, Date.
  • Varchar: Variable Character - series of numbers, letters and other characters.
  • Number: A number that can be used in calculations.
  • Date: Typically date/month/year format.
  • CSV: Comma Separated Value - file type for storing flat file databases.
  • Relational Databases: Stores and provides access to data points related to one another.
  • Problems with Single Tables: Duplicate data, wasted time and storage space, hard to extract specific data.
  • Primary Key: Unique identifier generated to reference records.
  • Foreign Key: Attribute created to link different tables.
  • ERD: Entity Relationship Diagram - Used to visualize relationships between entities (tables).

HTML

  • HTML: Markup language.
  • Tags: Always grounded by anchor brackets <>. Closed using /.
  • Metadata: Information stored inside the <body> tag about the website.
  • Attribute: Provides extra info about an element (e.g. class/id).
  • Comments: Done using <!-- content -->
  • Hyperlink: Links two pages together using anchor element <a>.
  • Absolute URL: Takes you to another URL.
  • Relative URL: Takes you to another page of the website you're on.

CSS

  • CSS: Cascading Style Sheets.
  • Purpose: Provides instructions on how HTML should display its content.
  • Proposed by: Håkon Wium Lee in 1994.

JavaScript

  • JavaScript: Created by Brendan Eich in 1995.
  • Purpose: Used to make a webpage interactive.
  • Implementation: Introduced into a website using the <script> tag.
  • DOM: Document Object Model - A model of all elements in an HTML website used by JavaScript to trigger functions.
  • Capabilities: Can be used to modify HTML and CSS elements.
  • Operators: Same as Python except ++ and -- for increment/decrement instead of += and -=.
  • Boolean Operators: && (And), || (Or), ! (Not).
  • Variable Assignment: var age=18
  • Lists: Assigned same as Python using var syntax.
